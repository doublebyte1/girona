Estrutura

Elevado numero de componentes / paquetes

Geoserver costumizado para geonode (autentificación de usuario)

http://build.geonode.org/geoserver/latest/

Hacer un ambiente para geonode

						
						$ cd /home/geonode/dev
						$ export VIRTUALENVWRAPPER_PYTHON=/usr/bin/python
						$ export WORKON_HOME=/home/geonode/dev/.venvs
						$ source /usr/local/bin/virtualenvwrapper.sh
						$ export PIP_DOWNLOAD_CACHE=$HOME/.pip-downloads
Y despues un git clone y installar los paquetes de python

Un geoserver especial

http://build.geonode.org/geoserver/latest/

El geonode (Django) asegura las permissiones de accesso a las capas en geoserver

Si el geoserver no se conecta al geonode, todas las capas se quedan 'cerradas'

El geonode solo usa un namespace de geoserver

Todas las capas en un namespace

Capas en outro namespace, no hay acesso

En los mailing list hay muchas dudas sobre la configuracion de Geoserver

Muchos se olvidam de configurar:

						
						date/security/auth/geonodeAuthProvider/config.xml

config.xml tiene que tener el IP de geonode (default:localhost)
